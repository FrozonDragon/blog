Reading Note for Linux Bible: Shell Enviroment

Bash Configuration Files

File for Whom When Executed What Info Remarks
/etc/profile every user log in path; location of your mailbox ; the size of your history files Gathers shell settings from configuration files in the /etc/profile.d directory.
/etc/bashrc every user each time a bash shell is opened sets the default prompt ; may add one or more aliases. Values in this file can be overridden by information in each user’s ~/.bashrc file.
~/.bash_profile each user user logs in and executed only once information that is specific to his or her own use of the shell By default it sets a few environment variables and executes the user’s .bashrc file.
~/.bashrc each user log in and also each time you open a new bash shell information that is specific to your bash shells. This is the best location to add environment variables and aliases so that your shell picks them up.
~/.bash_logout each user Executes each time you log out By default, it simply clears your screen.

Setting Your Prompt

  • The PS1 environment variable sets what the prompt contains.
  • If your shell requires additional input, it uses the values of PS2, PS3, and PS4.
  • To setting your prompt temporarily, type export PS1="[\t \w]\$",
    • The export command says to export the value to the shell so that it can be propagated to other shells you may open.
  • To make a change to your prompt permanent, add the value of PS1 to your .bashrc file in your home director

Characters to Add Information to bash Prompt

Special Character Meaning
\! Shows the current command history number. This includes all previous commands stored for your username.
\# Shows the command number of the current command. This includes only the commands for the active shell.
\$ Shows the user prompt ($) or root prompt (#), depending on which user you are.
\W Shows only the current working directory base name. For example, if the current working directory was /var/spool/mail, this value simply appears as mail.
\[ Precedes a sequence of nonprinting characters. This can be used to add a terminal control sequence into the prompt for such things as changing colors, adding blink effects, or making characters bold. (Your terminal determines the exact sequences available.)
\] Follows a sequence of nonprinting characters.
\\ Shows a backslash.
\d Displays the day name, month, and day number of the current date. For example: Sat Jan 23.
\h Shows the host name of the computer running the shell.
\n Causes a newline to occur.
\nnn Shows the character that relates to the octal number replacing nnn.
\s Displays the current shell name. For the bash shell, the value would be bash.
\t Prints the current time in hours, minutes, and seconds (for example, 10:14:39).
\u Prints your current username.
\w Displays the full path to the current working directory.

Starting Background Processes

  • To place a program in the background at the time you run the program, type an ampersand (&) at the end of the command line
  • The plus sign (+) shows that it was most recently placed in the background.
  • The minus sign (-) shows that it was placed in the background just before the most recent background job.
  • Use fg %n to bring any of the commands on the jobs list to the foreground.
  • If a command is stopped, you can start it running again in the background using the bg command.

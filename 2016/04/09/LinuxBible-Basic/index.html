Reading Note for Linux Bible: Basic

Features in Linux

  • No constant rebooting
  • Start/stop services without interrupting others
  • Portable software
  • Downloadable applications
  • No settings hidden in code or registries
  • Mature desktop
  • Freedom

Starting a Shell

Shell is the program used to interpret and manage commands.

Bash Shell is the default shell for most linux, it stands for Bourne Again Shell. The name is derived from the fact that bash is compatible with the first UNIX shell: the Bourne shell (represented by the sh command).

Other common shell

  1. C shell (csh), which is popular among BSD UNIX users
  2. Korn shell (ksh), which is popular among UNIX System V users.

Shell Prompt

  • $: a regular user
  • #: a root user

Practic 3: Checking Your Login Session

  • -u is to add information about idle time and the process ID
  • -H asks that a header be printed
  • tty1 is the monitor connected to the computer
  • pts/0 is telling you which “psuedo terminal” the user is logged in on, here is “putty” console
  • PID shows the process ID of the user’s login shell (the dot indicates that it is currently active)
  • COMMENT would show the name of the remote computer the user had logged in from

Practic 3: Checking Directories and Permissions

  1. Use pwd to check where you are
  2. Use echo $HOME to know your home directory
  3. Use cd or cd ~ to directly change directory to your home
  1. -a option to view all files including hidden files (dot files)
  2. -l option to see a long, detailed list of information on each file
  3. (.) means current directory
  4. (..) means parent directory

The number of characters shown for a directory (4096 bytes in these examples) reflects the size of the file containing information about the directory. While this number can grow above 4096 bytes for a directory that contains a lot of files, this number doesn’t reflect the size of files contained in that directory.

Checking System Activity

  1. -a option asks to show processes of all users who are associated with your current terminal
  2. -u option asks that usernames be shown, as well as other information such as the time the process started and memory and CPU usage
  3. STAT column represents the state of the process, with R indicating a currently running process and S representing a sleeping process.
  4. VSZ (virtual set size) shows the size of the image process (in kilobytes)
  5. RSS (resident set size) shows the size of the program in memory.
  6. START shows the time the process began running
  7. TIME shows the cumulative system time used.(Many commands consume very little CPU time, as is reflected by 0:00 for
    processes that haven’t even used a whole second of CPU time.)

Exiting the Shell

To exit the shell when you are done, type exit or press Ctrl+D.
